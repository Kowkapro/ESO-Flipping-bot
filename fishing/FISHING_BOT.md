# Fishing Bot — Документация и план разработки

> **Статус:** Код написан, ожидает тестирования в игре
> **Основной файл:** `fishing/main.py`
> **Альтернативный:** `fishing/yolo_fisher.py` (модульный, класс-based)

---

## Что делает бот

Автоматически рыбачит в ESO: ходит между рыболовными лунками, ловит рыбу, переходит к следующей лунке. Работает полностью визуально — через YOLO нейросеть "смотрит" на экран игры.

### Полный цикл (одна итерация)

```
1. Открыть карту (M)
2. Приблизить карту (10 кликов на "+")
3. YOLO-скан: найти синие крючки (blue_hook) на карте
4. Отфильтровать уже посещённые лунки
5. Выбрать ближайшую к игроку -> поставить метку (F) -> закрыть карту

6. Повернуть камеру к метке на компасе (Phase B)
7. Бежать спринтом к метке, корректируя курс по компасу (Phase C)
8. Определить прибытие (маркер исчез / прыгнул / interaction_prompt)

9. Осмотреться (4x90 поворот), найти лунку (interaction_prompt / bubbles)
10. Рыбачить: заброс (E) -> ждать крючок -> подсечка (E) -> лут (R) -> повтор
11. Лунка иссякла (2 пустых заброса) -> перейти к шагу 1
```

---

## Архитектура (4 фазы)

### Phase A — Выбор лунки (карта)
- Открываем карту, зумим на максимум
- YOLO находит все `blue_hook` иконки
- Дедупликация (лунки ближе 30px = одна и та же)
- Фильтрация уже посещённых (хранятся как dx,dy от центра экрана)
- Кликаем на ближайшую, ставим метку (F), закрываем карту

**Ключевые параметры:**
- `MAP_ZOOM_CLICKS = 10` — количество кликов зума
- `HOOK_MIN_CONF = 0.3` — минимальная уверенность YOLO
- `HOOK_DEDUP_DIST = 30px` — порог дедупликации
- `VISITED_DEDUP_DIST = 40px` — порог "уже посещена"

### Phase B — Поворот к метке
- YOLO ищет `compass_marker` на полоске компаса
- Рассчитывает смещение от центра экрана
- Плавно поворачивает камеру (human_mouse_arc, S-curve, 500Hz)
- До 20 попыток выровняться

**Ключевые параметры:**
- `MARKER_MIN_CONF = 0.3`
- `SCREEN_TO_MOUSE = 2.42` — конвертация пикселей экрана в пиксели мыши
- `DEAD_ZONE_FRAC = 0.02` — "мёртвая зона" (считается выровненным)
- `STEER_DAMPING = 0.9` — демпфирование (чтобы не перелетать)

### Phase C — Бег к метке
- Спринт (W + Shift) с коррекцией курса по компасу
- Каждые 150мс: YOLO-скан -> проверка маркера -> коррекция

**Определение прибытия (по приоритету):**
1. `interaction_prompt` обнаружен -> ARRIVED (мы у лунки)
2. Маркер прыгнул >= 250px за 1 кадр -> ARRIVED (прошли мимо метки)
3. Маркер потерян 8+ кадров подряд -> ARRIVED (подошли вплотную)
4. Таймаут 60 сек -> TIMEOUT

**Антизастревание:**
- Каждые 2 сек проверяет: сместился ли маркер хотя бы на 15px
- Если нет -> прыжки (2-3 раза)

**Ключевые параметры:**
- `RUN_STEER_DAMPING = 0.7`
- `RUN_MAX_DURATION = 60.0s`
- `MARKER_JUMP_THRESHOLD = 250px`
- `MARKER_LOST_THRESHOLD = 8` кадров
- `STUCK_CHECK_INTERVAL = 2.0s`

### Phase D — Рыбалка
- Заброс (E) -> ждать белый крючок (Votan's Fisherman) -> подсечка (E) -> лут (R)
- 2 пустых заброса подряд = лунка иссякла
- Все задержки рандомизированы (имитация человека)

**Ключевые параметры:**
- `MAX_WAIT_FOR_HOOK = 45s` — макс ожидание клёва
- `MAX_FAILED_CASTS = 2` — пустых забросов до "иссякла"
- `HOOK_WHITE_THRESHOLD = 220` — яркость пикселя для определения крючка
- `HOOK_WHITE_RATIO = 0.08` — доля белых пикселей

## YOLO Модель

- **Архитектура:** YOLOv8s (small)
- **Путь:** `fishing/training/runs/eso_fishing/weights/best.pt`
- **Производительность:** ~20 FPS inference, ESO остаётся на 100 FPS
- **Обучение:** 231 изображений, 468 аннотаций, mAP50=0.879

### Классы (8 штук)

| ID | Класс | Аннотаций | mAP50 | Для чего |
|----|-------|-----------|-------|----------|
| 0 | blue_hook | 109 | OK | Иконка лунки на карте |
| 1 | bubbles | 66 | 0.359 | Пузыри у лунки (слабый!) |
| 2 | compass_marker | 165 | OK | Метка на компасе |
| 3 | enemy | 54 | OK | Враг (для будущего) |
| 4 | interaction_prompt | 28 | 0.995 | "[E] Ловить рыбу" |
| 5 | hp_bar | 9 | - | HP бар (мало данных) |
| 6 | xp_popup | 2 | - | XP попап (мало данных) |
| 7 | destination_text | 35 | OK | Текст назначения |

### Слабые классы (нужно больше данных)
- `bubbles` — 0.359 mAP50, ложные срабатывания при беге. Нужно 50+ аннотаций
- `hp_bar` — только 9 аннотаций, нужно 20+
- `xp_popup` — только 2 аннотации, нужно 20+

## Управление ботом

- **F5** — Старт (сначала переключиться на ESO!)
- **F6** — Стоп (отпускает все клавиши)

### Запуск
```bash
python fishing/main.py
```

### Предварительные требования
- ESO в оконном полноэкранном режиме (1920x1080)
- Аддон Votan's Fisherman включён (показывает белый крючок)
- Аддон HarvestMap включён (показывает синие крючки на карте)
- YOLO модель обучена (`fishing/training/runs/eso_fishing/weights/best.pt`)
- Запуск от администратора (для keyboard hooks)

| Параметр | Значение | Как получено |

## Известные проблемы

| Проблема | Влияние | Решение |
|----------|---------|---------|
| `bubbles` слабый (mAP50=0.359) | Ложные срабатывания при беге | Не использовать как arrival signal, нужно 50+ аннотаций |
| VISITED_DEDUP_DIST=40px не проверен | Может пропускать/дублировать лунки | Настроить после тестирования |
| Per-frame logging в Phase C | Много вывода в консоль | Уменьшить до каждого 5-го кадра |
| Нет Telegram-уведомлений | Не знаешь когда бот остановился | Добавить после тестирования |
| Нет управления инвентарём | Бот остановится когда полный | Добавить позже |
| Маркер не виден если лунка рядом | Компас не показывает метку вблизи | look_around решает это |
